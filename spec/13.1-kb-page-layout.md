# Spec 13.1: KB Page Layout

**Status:** Complete
**Created:** 2025-12-01
**Parent:** [Spec 13: Lightweight Knowledge Base](13-lightweight-knowledge-base.md)

## Overview

Create the basic KB page at `/kb` with navigation link and basic layout structure.

## Requirements

### Functional Requirements

1. **Navigation**
   - Add "Knowledge Base" link to main navigation/header
   - Link navigates to `/kb` route

2. **Page Layout**
   - Page title: "Knowledge Base"
   - Empty state message when no documents exist
   - Consistent styling with existing pages (Ideas page)

3. **Schema**
   - Create KBDocument model in Amplify schema
   - Fields: id, title, content, createdAt, updatedAt

### Technical Requirements

1. **Route** - `/kb` using Next.js App Router
2. **Components** - Server component for initial render
3. **Data** - Fetch documents from DynamoDB via Amplify

## Implementation Plan

### Files to Create

1. **src/app/kb/page.tsx** - Main KB page
2. **amplify/data/resource.ts** - Add KBDocument model

### Files to Modify

1. **src/app/page.tsx** or layout - Add navigation link to KB

## Test Cases

### Cypress E2E Tests

**Test 1: KB Page Exists**
```typescript
it("displays the KB page", () => {
  cy.visit("/kb");
  cy.contains("h1", "Knowledge Base").should("be.visible");
});
```

**Test 2: Empty State**
```typescript
it("shows empty state when no documents", () => {
  cy.visit("/kb");
  cy.get('[data-testid="kb-empty-state"]').should("exist");
});
```

**Test 3: Navigation Link**
```typescript
it("has navigation link to KB page", () => {
  cy.visit("/");
  cy.contains("a", "Knowledge Base").click();
  cy.url().should("include", "/kb");
});
```
